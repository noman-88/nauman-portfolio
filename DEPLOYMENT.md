# 🎯 Deployment & Production Guide\n\n## Pre-Deployment Checklist\n\n### Code Quality\n- [ ] No console errors in development\n- [ ] All components working correctly\n- [ ] Animations smooth and performant\n- [ ] Contact form tested and working\n- [ ] All links functional\n- [ ] Responsive on mobile/tablet/desktop\n- [ ] No broken images\n- [ ] API endpoints tested\n\n### Configuration\n- [ ] `.env` file configured correctly\n- [ ] MongoDB database ready\n- [ ] CORS settings appropriate for domain\n- [ ] Environment variables secured\n- [ ] No hardcoded secrets\n\n### Performance\n- [ ] Images optimized\n- [ ] Bundle size acceptable\n- [ ] Lazy loading implemented where needed\n- [ ] Cache headers configured\n- [ ] API response times acceptable\n\n---\n\n## 🌐 Deployment Options\n\n### Option 1: Vercel (Frontend)\n\n**Best for**: React frontend with Vite\n\n```bash\n# Install Vercel CLI\nnpm i -g vercel\n\n# Navigate to client\ncd client\n\n# Deploy\nvercel\n```\n\n**Configuration**:\n- Framework: Vite\n- Build Command: `npm run build`\n- Output Directory: `dist`\n\n---\n\n### Option 2: Netlify (Frontend)\n\n**Best for**: React frontend with Vite\n\n1. Connect GitHub repository\n2. Set build command: `cd client && npm run build`\n3. Set publish directory: `client/dist`\n4. Deploy\n\n---\n\n### Option 3: Heroku (Backend)\n\n**Best for**: Express API server\n\n```bash\n# Install Heroku CLI\nnpm install -g heroku\n\n# Login\nheroku login\n\n# Create app\nheroku create your-app-name\n\n# Set environment variables\nheroku config:set MONGO_URI=your_mongo_uri\nheroku config:set PORT=5000\n\n# Deploy\ngit push heroku main\n```\n\n---\n\n### Option 4: Railway (Backend)\n\n**Best for**: Express API server (newer alternative to Heroku)\n\n1. Connect GitHub repository at railway.app\n2. Set environment variables in dashboard\n3. Deploy automatically on push\n\n---\n\n### Option 5: Render (Backend)\n\n**Best for**: Express API server\n\n1. Connect GitHub at render.com\n2. Create Web Service\n3. Set build command: `npm install`\n4. Set start command: `node server/server.js`\n5. Add environment variables\n6. Deploy\n\n---\n\n## 🔗 Full Stack Deployment\n\n### Using Vercel + Render\n\n1. **Deploy Backend to Render**\n   - Push server to GitHub\n   - Create service on Render\n   - Get API URL: `https://your-api.onrender.com`\n\n2. **Update Client API URL**\n   ```javascript\n   // In axios calls, use environment variable\n   const API_URL = process.env.REACT_APP_API_URL || 'http://localhost:5000';\n   ```\n\n3. **Create .env.production**\n   ```\n   REACT_APP_API_URL=https://your-api.onrender.com\n   ```\n\n4. **Deploy Frontend to Vercel**\n   - Push to GitHub\n   - Connect to Vercel\n   - Set environment variable in Vercel dashboard\n   - Deploy\n\n---\n\n## 🔐 Security Checklist\n\n- [ ] CORS properly configured\n- [ ] No sensitive data in frontend\n- [ ] API rate limiting enabled\n- [ ] Input validation on server\n- [ ] HTTPS enforced\n- [ ] Environment variables not committed\n- [ ] Database credentials secured\n- [ ] API keys rotated regularly\n\n---\n\n## 📊 Performance Optimization\n\n### Frontend\n```bash\n# Build and check size\nnpm run build\n# Look for bundle size recommendations\n```\n\n### Backend\n- Enable compression\n- Use caching headers\n- Implement pagination for APIs\n- Monitor query performance\n\n---\n\n## 🔄 Continuous Integration\n\n### GitHub Actions Example\n\n```yaml\nname: Deploy\n\non:\n  push:\n    branches: [main]\n\njobs:\n  deploy:\n    runs-on: ubuntu-latest\n    steps:\n      - uses: actions/checkout@v2\n      - uses: actions/setup-node@v2\n        with:\n          node-version: '18'\n      - run: npm install\n      - run: cd client && npm run build\n      - name: Deploy\n        run: npm run deploy\n```\n\n---\n\n## 📈 Monitoring & Analytics\n\n### Recommended Tools\n- **Error Tracking**: Sentry\n- **Analytics**: Google Analytics, Vercel Analytics\n- **Performance**: New Relic, DataDog\n- **Logs**: LogRocket, Papertrail\n\n---\n\n## 🚨 Troubleshooting Deployment\n\n### CORS Errors\n```javascript\n// server.js\napp.use(cors({\n  origin: 'https://your-domain.com',\n  credentials: true\n}));\n```\n\n### Database Connection Issues\n```javascript\n// Check MongoDB Atlas IP whitelist\n// Verify connection string format\n```\n\n### Build Failures\n```bash\n# Clear cache and rebuild\nrm -rf node_modules package-lock.json\nnpm install\nnpm run build\n```\n\n---\n\n## 📞 Post-Deployment\n\n1. **Test all features**\n   -
